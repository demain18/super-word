#### 1\. 기획

지금부터 만들건 '슈퍼워드' 라는 이름의 직장인의 업무를 돕기 위한 보고서 자동 생성 및 내용 작성 웹서비스야.

외근 보고서, 출장 보고서, 회의록, 주간/월간 업무보고서, 실적 보고서. 이렇게 5개 종류의 보고서를 Word 형태로 생성하는 기능이 핵심이고

Claude api를 사용하여 docx skill을 활용해 워드 파일을 작성할거고, 백엔드에서 생성된 해당 양식 파일을 사용자에게 preview 형식으로 보여주기 위해서 pdf로 변환 후 화면에 출력하는 방식으로 실시간 미리보기 기능을 구현해.



슈퍼워드는 3단계로 서비스를 제공해야 해.



1단계) 사용자가 5개의 보고서 양식중 하나를 선택한다.

2단계) 로딩을 걸쳐 생성된 보고서 양식을 보고 (pdf preview방식) 사용자가 더 전통적인 디자인으로, 더 트렌디하고 글로벌한 디자인으로 와 같은 선택지 (글로써 피드백을 전달하는 서술형 선택지도 제공)중에 하나를 클릭하고 다시 로딩한 뒤 결과물에 만족했다면 '다음 단계로 넘어가기' 버튼을 클릭

3단계) 여기에서는 사용자가 양식을 보고 그 안에 필수적인 정보 (이름, 외근 이유, 외근 장소 이런것들을 서술형으로 작성) 를 입력하면 이를 기반으로 재구성하여 양식을 채워넣음. 만족했다면 '완료하기' 버튼을 누르고 완성된 양식 파일을 다운로드 받음



물론 이 과정중에 2레벨에서도 다운받을 수 있어야 해. 모든 작업이 완료되어야지 다운받을 수 있는게 아니고, 양식만 작성하거나 양식을 받아서 그 다음 한번 수정한 상태거나 이럴때도 모두 다운로드가 항시 가능하도록 버튼을 배치해.



#### 2\. 기능

* 기본적으로 상단에 Nav  바를 두고 여기에서 3단계를 진행시키며 상단 바 좌측에는 슈퍼워드의 로고가 들어감. 하단 메인 화면에서는 좌측에 preview창이 나오고 우측에는 사용자가 선택할 수 있는 선택지들이 나오며 그중 서형으로 피드백도 가능해야 함 (기본적으로 버튼만 눌러서 피드백하는게 편하니까)
* 워드 파일 생성에는 docx skill을 사용할것이며 claude api를 연결해 llm 엔진으로써 채택함
* 파일 생성 과정에서는 preview  ui부분이 전체적으로 가우시안 블러가 들어가면서 파일이 수정되는 것처럼 연출되어야 하고, 실제로는 백엔드 단에서 생성/수정한 문서를 pdf로 출력하여 바꿔치기 하는 방식으로 작동함. 어떤어떤 수정을 하고있어요 하면서 로딩 애니메이션이 나와야 함
* 2단계인 양식 생성 단계에서는 다음과 같은 선택지가 존재함. 대기업 스타일, 글로벌 스타트업 스타일, 공무원/정부 스타일 이렇게 선택할 수 있음. 선택지를 누를때마다 수정 프롬프트에 조금씩 더 (한 15% 정도) 해당 스타일에 대한 피드백과 개선 요청이 들어감.
* 기본적으로 AI의 페르소나를 설정하여 프롬프트에 삽입할 것. 페르소나의 경우 별개의 파일인 Persona.md을 읽고 이에 맞추어 개발할 것
* 3단계인 양식 채우기에서는 우측에 선택지 부문에서 완전히 서술형으로 내 이름이나 대체로 필요한 부분들을 알아서 던져주면 그걸 기반으로 ai로 안에 채워넣는 방식. 글을 쓸때 페르소나에 맞게 실제로 직장인이 보고서를 작성하는 것처럼 어색하지 않고 자연스러우면서 사무적인 어투로 작성되어야 함. 여기는 클로드나 제미나이를 쓸때처럼 대화형으로 피드백을 주고받음. 사용자가 서술형으로 텍스트로 요구사항을 전달할것임.
* 백엔드 개발의 경우 supabase를 사용할것이며 Cowork에서 생성하는 문서 양식 수준의 퀄리티를 목표로 함



#### 3\. 기술 스택

* React, Next.js, Typescript, Emotion.css, 반응형 웹 (pc\&mobile 지원), Vercel
* Supabase (edge function, etc), Claude docx-skill



#### 4\. 요구 사양

* pc, tablet, mobile 화면에서 모두 반응형으로 작동하는 UI를 제공해야 함
* 양식 생성 및 양식 채우기 작업은 초기 최초 생성 단계에서는 최대 1분, 수정의 경우 30초 를 넘기지 않아야 함.  수정 시간 목표는 20초 내외임.
